Chapter 0: Comments on how I approached the challenge in general

  • I worked in a branch, so I can relax project disciplin (e.g.: allow myself to make uncontained or incomplete commits by mistake to safe reviewing time)
  • I disabled Neurocid features that were in the way and are not strictly necessary (e.g.: like compiling to javascript using emscripten)
  • Where possible I ignored Neurocid issues.
  • I didn't fix issues/warnings in third-party code (e.g.: Box2D).
  • I held back on beautifying the App and the Docs because you are looking for a backend hacker.
  • I made sure the project can be edited, build and run with and without QT-Creator.
  • I didn't test and benchmark to a professional extent but i made sure it runs on Ubuntu, Debian and OpenSUSE.
  • I didn't test for Mac OSX at all (which is a supported target of the master-branch)
  • I chose a quite ambigous task knowing it is significantly more work than required, because i wanted to implement something that wouldn't go to waist, no matter what.
  • In general it would be very nice if you could provide a full Felgo-qdoc-tutorial with config a qdoc document, style-sheets and assets.
  • Most of the time I'm not a last minute guy but job, family and virus-life took their toll.
  • I didn't include the appendix (just) to show off. If focus/given directions/time allow(s) it, I regard doing a tutorial as testing a tutorial (in projects I am involved in)
  • I didn't include the qt/qtdoc pitfalls that i encountered. e.g.: it took quite some time to find out that source documentation can't be in source files and doesn't warn about it. try to search for it :)
