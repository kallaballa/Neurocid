Chapter 1: Building Neurocid for Debian Linux 10

This chapter assumes a minimal Debian Linux 10 install as prerequisite.

First, for convenience we are going to install sudo and configure it:

su - # change to root
apt-get install sudo
adduser <your user> sudo

Please note, that you need to log out and back in for the changes to take effect (and hence being able to use sudo) From this point all commands that are not prefixed with "sudo" should be run with the unprivileged user that you want to run felgo-neurocid with.

Now, let's install all build dependencies:

sudo apt-get install git-core pkg-config cmake build-essential g++ libsdl1.2-dev libsdl-gfx1.2-dev libsdl-image1.2-dev libsdl-ttf2.0-dev libboost-system-dev libboost-program-options-dev libboost-serialization-dev libsndfile1-dev

Next let's get the code. We are going to clone the "felgo" branch (not master!).

git clone --branch felgo https://github.com/kallaballa/Neurocid

Now we need to install Felgo. Please follow the corresponding tutorial and make sure to install Felgo directly in your home folder (/home/<your user>/Felgo) or to adjust the variable "FELGO_PATH" in the main Neurocid Makefile (Neurocid/Makefile). For example, if you installed Felgo to /opt/Felgo you need to change the variable to the following:

FELGO_PATH :=/opt/Felgo/

Now we are ready to build Neurocid (including the felgo frontend)!

cd Neurocid/ # change to the Neurocid directory

Start by building the third-party code of the repository:

./build_deps.sh

And now let's actually build neurocid including the felgo qt-project which resides in ./felgo/

make -j4 # change 4 to the number of cores/hyperthreads/CPUS your system has to accelerate the build process. if you don't want to do a parallel build, just omit '-j4'

And don't forget to install!

sudo make install
